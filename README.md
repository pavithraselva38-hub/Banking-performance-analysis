# Banking-performance-analysis
End-to-end Banking Analytics project using MySQL &amp; Power BI with SQL analysis, interactive dashboards, KPI reporting, and business recommendations.

**#Banking Analytics using SQL & Power BI**
**Project Overview**

This project demonstrates an end-to-end Banking Analytics solution using MySQL and Power BI. The objective was to transform raw banking data into actionable business insights through data validation, SQL analysis, and interactive dashboards.

The analysis focuses on:
•	Customer Analytics
•	Account Analytics
•	Transaction Analytics
•	Merchant Analytics
•	Loan Analytics
•	Executive Banking Performance
Project Objectives
•	Perform data validation and quality assessment.
•	Generate business KPIs using SQL.
•	Analyze customer, transaction, merchant, and loan performance.
•	Develop interactive Power BI dashboards.
•	Provide business recommendations supported by analytical findings.

**Dataset**:
The project consists of 7 relational datasets.

Relationship	Description
Customers → Accounts	One customer can own one or more bank accounts.
Accounts → Transactions	Each account can have multiple financial transactions.
Transactions → Merchants	Every transaction is associated with a merchant.
Customers → Loans	A customer may have one or more loans.
Accounts → Cards	A bank account may have one or more associated cards.
Branches->	Stores branch information independently and serves as a reference dataset in this project.

**Key Business Insights:**
•	Customer registrations fluctuated over the years.
•	All credit score segments dominated the customer portfolio except for exceptional category.
•	February consistently showed lower transaction activity.
•	High-value merchants contributed significantly to transaction value.
•	Loans above 11% interest formed the largest interest rate segment.

**Business Recommendations:**
•	Improve customer acquisition.
•	Strengthen customer retention.
•	Optimize merchant partnerships.
•	Monitor lending exposure.
•	Improve transaction engagement.
